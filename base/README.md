1) **«Помогите Даше решить проблему»** — по сути это самая базовая задача: прочитать два целых числа A и B и вывести их сумму A+B. Сложность минимальная: главное не ошибиться с типом данных (нужен 64-bit, чтобы не переполниться в языках с int).

2) **«Покупка друзей»** — дано количество конфет A и орешков B, нужно найти максимальное число ребят, которым можно раздать поровну и конфеты, и орешки без остатка (каждому по y конфет и x орешков). Суть сводится к поиску максимального общего делителя. Cложность в том, чтобы быстро и корректно посчитать НОД на больших числах до 10^9.

3) **«Студенты всегда хотят спать»** — нужно найти первый индекс i (нумерация с 0), для которого сумма элементов слева от i равна сумме элементов справа от i; если такого нет — вывести -1. Основная сложность в ограничениях, поэтому решение должно быть строго линейным O(N) и аккуратным по памяти/вводу (обычно делают через общую сумму и префикс, проходом в один-два раза).

4) **«Сдвинь меня, если сможешь»** — требуется выполнить циклический сдвиг массива на K позиций (вправо при K>0, влево при K<0) внутри массива, без дополнительной памяти, и вывести получившуюся последовательность. Сложность — правильно нормализовать очень большой |K| (до 10^9) по модулю N и реализовать in-place сдвиг (через развороты или циклы по НОД), не путая направления.

5) **«Гений трейдинга»** — нужно посчитать, сколько чисел n в диапазоне [X,Y] (до 10^18) имеют в двоичной записи ровно 3 группы подряд идущих одинаковых битов (то есть 3 “пробега” вида 111…000…111…, и т.п.). Сложность в огромном диапазоне: перебор невозможен, обычно нужна двоичная “цифровая динамика” (DP по битам) с учётом количества групп и последнего бита.

6) **«Достойное решение проблемы [доп. балл]»** — есть N комнат по кругу, каждая с числом x_i; нужно поставить перегородки между соседними комнатами, разбив круг на блоки, так чтобы внутри любого блока тараканы “не размножались”, то есть для любых двух комнат блока выполнялось условие, что нет общих установленных битов. Требуется минимизировать число блоков (при этом хотя бы одна перегородка должна быть), а сложность — в круговой структуре и больших ограничениях, нужно эффективно поддерживать условие “без пересечений по битам” и учитывать, что выбор места разреза по кругу влияет на оптимум.
