1. **1984** — по строкам `T` (текст) и `X` (шаблон) нужно найти количество вхождений `X` в `T` и вывести все индексы начала этих вхождений (с 0), при длинах до `2·10^5`. Сложность — сделать это за линейное/почти линейное время (КМП / Z-функция / префикс-функция), а не наивно `O(|T|·|X|)`.

2. **Хогвартс** — для каждой позиции строки нужно вывести максимальное `N`, такое что некоторый суффикс префикса `s[0..i]` совпадает с префиксом всей строки (то есть длину наибольшей границы префикса) — фактически массив значений префикс-функции/бордеров по всем префиксам. Сложность — корректно посчитать префикс-функцию (или Z-функцию) за `O(n)` и вывести значения для всех букв.

3. **Лиана Клубина** — дан текст длины `N` и `M` запросов вида `(i, j, k)`: проверить, равны ли две подстроки длины `k`, начинающиеся в `i` и `j`, и посчитать, сколько раз фанаты “оказались правы”. Сложность — отвечать на `M` запросов быстро при `N` до `10^6`: обычно делают полиномиальные хеши (лучше двойные) или суффиксные структуры, чтобы сравнение подстрок было `O(1)`.

4. **Junior-библиотекарь** — поддерживать отсортированный по имени список авторов: приходят запросы либо добавить новое имя, либо вывести имя автора, который сейчас стоит на позиции `k` в отсортированном списке. Сложность — нужны динамические операции “вставка + k-й элемент” (order statistics), поэтому используют дерево с порядковыми статистиками/Фенвика по сжатым координатам или treap; важно уложиться в `N ≤ 10^5` операций.

5. **Секретная служба** — для каждого из `M` жителей нужно проверить корректность записи персональных данных по строгому формату (ФИО из трёх слов с заглавной буквы и латиницей, возраст — ровно две цифры, телефон — 11 цифр с допустимыми вариантами записи/разделителями, адрес с заданными префиксами и правилами для города/улицы/дома) и вывести `YES/NO`. Сложность — аккуратно разобрать строку и провалидировать её по набору правил (по сути “ручной парсер/регэкспы”), не пропуская пограничные случаи и не путая допустимые символы/позиции.
