1. **Что же такое множество?** — нужно проверить равенство двух множеств размера `N` по их элементам, причём сортировки и готовые контейнеры/деревья запрещены, сравнение предлагается делать “через хеширование”. Сложность — придумать корректный (желательно двойной) хеш множества, который не зависит от порядка ввода и устойчив к коллизиям, а затем за `O(N)` сравнить два набора при `N` до `10^6`.

2. **Тест** — поддерживать динамические группы студентов с операциями: получить средний балл группы, удалить студента, добавить студента с баллами, и найти максимальный балл в группе, при огромном числе запросов (`Q` до `10^7`) и запрете на стандартные структуры (`set/map/unordered_*`). Сложность — спроектировать свою структуру (обычно хеш-таблица для поиска по ISU + куча/счётчики для max + поддержка суммы/размера для среднего) так, чтобы все операции были амортизированно быстрыми и без просадок по памяти.

3. **Биба и Боба** — посчитать количество пар индексов `(i, j)`, для которых выполнено `S[j] − S[i] = j − i` (разность высот равна числу стульев между ними), при очень больших `N` и запретах на сортировку/деревья. Сложность — заметить преобразование `S[i] − i` и свести задачу к подсчёту равных значений через собственную хеш-таблицу частот за `O(N)`.

4. **Три друга 3: Кукушка** — по трём спискам уникальных имён файлов нужно начислить очки каждому студенту: 3 балла за уникальный файл, 1 — если файл встречается у ровно двоих, 0 — если у всех троих, причём требуется реализовать алгоритм “кукушки” (cuckoo hashing) вместо стандартных структур. Сложность — быстро хранить и сравнивать большое число строк без `unordered_map`, обеспечить стабильную работу хеш-таблицы и корректно агрегировать частоты появления файлов.

5. **Катин лес** — проверить, являются ли два неориентированных дерева изоморфными (можно ли перенумеровать вершины одного так, чтобы совпала смежность со вторым). Сложность — сделать это за `O(n)`–`O(n log n)` при `n` до `2·10^5`: обычно используют каноническое кодирование дерева (AHU) / хеширование поддеревьев от центров, чтобы сравнить “структурные хеши” деревьев, а не перебирать перестановки.
