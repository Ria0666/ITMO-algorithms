1. **Сортировка тапок (Insertion sort)** — нужно отсортировать `N` размеров тапок по возрастанию, причём по условию требуется реализовать именно сортировку вставками. Сложность тут не в идее, а в аккуратной реализации “по учебнику”: корректно сдвигать элементы, уложиться в `O(N^2)` (при `N ≤ 10^4` это ещё ок) и правильно вывести результат.

2. **Паттинсон** — дан массив весов (включая отрицательные) размера до `10^6`, его надо отсортировать по возрастанию, причём запрещены встроенные сортировки и “чужие” алгоритмы: нужна самостоятельная реализация QuickSort. Сложность — сделать её устойчивой на больших данных: быстрый ввод/вывод, хороший выбор опорного, контроль глубины рекурсии и защита от худших случаев.

3. **Любовь жестока** — дано до `10^6` палочек, каждую можно использовать максимум в одном прямоугольнике, при этом разрешено укоротить палочку на `1`; нужно получить максимальную суммарную площадь всех построенных прямоугольников. Сложность — найти правильную жадную стратегию: после сортировки грамотно формировать пары сторон с допуском `1`, не “портя” будущие прямоугольники, и при этом уложиться по времени.

4. **Anti-QuickSort** — по заданному `n` требуется вывести перестановку чисел от `1` до `n`, на которой QuickSort из условия выполнит максимальное число сравнений (подойдёт любая худшая перестановка). Сложность — не в коде, а в понимании конкретного псевдокода quicksort и в построении контртеста, который гарантированно загоняет алгоритм в худший сценарий.

5. **Рейтинг** — дан массив рейтингов, нужно посчитать число пар `(i < j)`, которые нарушают неубывание (то есть число инверсий), и вывести это количество; по условию требуется реализовать сортировку слиянием. Сложность — уйти от `O(N^2)` к `O(N log N)` через подсчёт инверсий при merge, использовать 64-bit (инверсий может быть очень много) и не упереться во ввод при `N ≤ 10^6`.

6. **Нейроночка** — для каждого из `m` фильмов надо вычислить рейтинг как `sum(a_i * x_i)` (по модулю `10^9 + 7`), затем вывести номера `k` фильмов с максимальным рейтингом (при равенстве — меньший номер). Сложность — в производительности на больших входах, корректной модульной арифметике и в эффективном выборе top-k с правильным tie-break.

7. **Непрерывное техническое образование** — есть `N` лабораторных с длительностью `T` и дедлайном `D`; все нужно выполнить, начиная с дня `0`, а баллы за работу равны `D - (время её фактического завершения)`; требуется максимизировать суммарные баллы. Сложность — свести задачу к оптимальному порядку выполнения (планированию) и реализовать решение эффективно при `N` до `10^6`, аккуратно считая большие префиксные суммы.

8. **Грязные игры (доп. балл)** — дан стартовый порядок лошадей и деньги владельцев; разрешены соседние обмены, где задний платит `1` тугрик переднему, и обмен возможен только если у платящего есть деньги; нужно выяснить, существует ли финальный порядок, где “богачи довольны” (впереди их лошади не окажется владельца беднее), и вывести финальные капиталы в порядке финиша либо `:(`. Сложность — в том, что обмены меняют деньги и создают ограничения достижимости, а `N` огромный (до `2 * 10^6`), поэтому нужна умная модель/жадность, а не прямой “пузырёк”.
