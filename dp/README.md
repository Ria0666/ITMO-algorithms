1. **Ромашки** — дано поле `n × m` с количеством цветов в каждой клетке и `q` точек встречи `(x_i, y_i)`; Маша идёт из `(0,0)` только вправо/вниз, Даша — из `(n−1,m−1)` только влево/вверх, и если девочка попадает в точку встречи, она “собирает” все цветы на своём пройденном участке; для каждой точки нужно вывести максимум собранных цветов. Сложность — быстро отвечать на до `10^6` запросов при `n·m ≤ 10^5`: обычно заранее считать префиксные/суффиксные суммы (или DP по путям) и сводить каждый запрос к `O(1)`-вычислению, аккуратно обрабатывая точки вне поля.

2. **Горы Краснодарского края** — по последовательности высот свай длины `n` нужно найти максимальную длину строго возрастающей подпоследовательности (канатной дороги). Сложность — `n` до `10^6`, поэтому нужен `O(n log n)` (классический LIS через массив “минимальных хвостов” и `lower_bound`), а не `O(n^2)`.

3. **Аварийный комплект без одного компонента** — есть `n` предметов с целыми весами и вещественными стоимостями и рюкзак вместимости `W`; для каждого `j` надо найти оптимальную суммарную стоимость при запрете брать предмет `j`. Сложность — сделать это быстрее, чем `n` раз решать полный 0/1 knapsack: применяют две динамики (префикс/суффикс по предметам) и “склейку” по весу за `O(n·W)` на построение и `O(W)` на ответ для каждого `j`, сохранив точность вывода.

4. **Серверные панели 3×1** — посчитать число способов полностью замостить прямоугольник `n × m` домино `3×1` (вертикально) и `1×3` (горизонтально) без наложений и выходов за границу. Сложность — ограничения маленькие по `n` (до 9), но `m` до 30, поэтому обычно используют DP по колонкам с битмасками (и иногда с состоянием на 2 колонки вперёд), аккуратно генерируя переходы.

5. **Рюкзак по ценности** — 0/1-рюкзак: веса вещественные, ценности целые; нужно максимизировать суммарную ценность при суммарном весе ≤ `W`. Сложность — из-за вещественных весов динамика по весу даёт неточности/неудобна, поэтому считают DP по суммарной ценности: `dp[v]=минимальный вес`, и берут максимальное `v`, для которого `dp[v] ≤ W`, следя за погрешностью сравнения.
